Pseudo Assembly Language Documentation

This pseudo-assembly language and interpreter were created based on lectures from the Warsaw University of Technology (Politechnika Warszawska, PW). The instruction set provides a simplified assembly-like environment for learning fundamental computer architecture concepts.

The language includes register-memory operations, register-register operations, conditional jumps, and data declaration instructions, all implemented with a little-endian memory model.

Register-Memory Instructions

Instruction Syntax Example Description
A A <reg>, <addr> A 1, LABEL Add memory value to register.
S S <reg>, <addr> S 2, 100 Subtract memory value from register.
M M <reg>, <addr> M 3, VALUE Multiply register by memory value.
D D <reg>, <addr> D 1, 0(2) Integer divide register by memory value.
C C <reg>, <addr> C 5, END Compare register with memory value.
L L <reg>, <addr> L 0, DATA Load memory value into register.
ST ST <reg>, <addr> ST 4, RESULT Store register value to memory.
LA LA <reg>, <addr> LA 6, ARRAY Load address into register.

Register-Register Instructions

Instruction Syntax Example Description
AR AR <reg1>, <reg2> AR 1, 2 Add reg2 to reg1.
SR SR <reg1>, <reg2> SR 3, 1 Subtract reg2 from reg1.
MR MR <reg1>, <reg2> MR 0, 4 Multiply reg1 by reg2.
DR DR <reg1>, <reg2> DR 5, 2 Integer divide reg1 by reg2.
CR CR <reg1>, <reg2> CR 1, 0 Compare reg1 with reg2.
LR LR <reg1>, <reg2> LR 3, 7 Load reg2 value into reg1.

Jump Instructions

Instruction Syntax Example Description
J J <addr> J LOOP Unconditional jump to address.
JP JP <addr> JP POSITIVE Jump if positive
JN JN <addr> JN NEGATIVE Jump if negative
JZ JZ <addr> JZ ZERO Jump if zero

Data Declaration Instructions

Instruction Syntax Example Description
DC DC INTEGER(<value>) DC 42 Define constant - allocates 4 bytes with initial value.
DC DC <count>*INTEGER(<value>) DC 5*INTEGER(10) Define multiple constants - allocates count*4 bytes, first initialized to value.
DS DS INTEGER DS INTEGER Define storage - allocates 4 bytes initialized to zero.
DS DS <count>*INTEGER DS 10*INTEGER Define storage - allocates count*4 bytes initialized to zero.

Addressing Modes

Memory Address Formats:

  • LABEL - Direct label reference
  • 123 - Direct numeric address
  • 0(5) - Indirect addressing using register 5 as pointer

Flags:

  • ZF (bit 6) - Zero Flag: Set when result equals zero
  • SF (bit 7) - Sign Flag: Set when result is negative

Registers:

16 general-purpose registers numbered 0-15, each storing 32-bit signed integers.

Memory Layout:

Little-endian byte ordering, 4-byte (32-bit) words, with data and instruction segments.
